## 0. 分析背景

本报告基于对 `src/models/` 目录下所有模型文件的深度代码分析，梳理项目中使用的机器学习模型、使用原因及设计架构。

---

## 1. 核心发现

项目采用**多层次模型架构**，以梯度提升树为主、深度学习可选、集成学习增强：

| 层级 | 模型类型 | 核心文件 | 应用场景 |
|------|---------|---------|---------|
| **主模型** | XGBoost/LightGBM | `predictor.py`, `stock_predictor.py` | 板块/个股涨跌幅预测 |
| **可选模型** | LSTM/Transformer | `deep_learning.py` | 时序模式捕捉 |
| **增强模型** | Stacking Ensemble | `model_optimizer.py` | 降低单一模型偏差 |
| **优化工具** | Optuna | `model_optimizer.py` | 超参数贝叶斯优化 |
| **运维机制** | Incremental Learner | `online_learning.py` | 增量更新+版本管理 |

## 2. 详细分析

### 2.1 梯度提升树模型（核心预测模型）

#### 使用原因

- **标准化特征 + 梯度提升树** = 业界最佳实践的量化预测方案
- XGBoost/LightGBM 对金融时序特征处理能力强
- 支持并行训练，适合大规模数据处理
- 特征重要性可解释性强

### 2.2 深度学习模型（LSTM/Transformer）

#### 使用原因

- **LSTM**：擅长捕捉长程时序依赖，适合识别趋势延续模式
- **Transformer**：注意力机制可捕捉多时间尺度模式
- 作为可选增强方案，不强制依赖 PyTorch

### 2.3 集成学习模型（Stacking）

#### 使用原因

- 多模型组合降低单一模型偏差
- Stacking 利用基模型互补性
- Ridge 元模型避免过拟合

### 2.4 超参数优化（Optuna）

#### 使用原因

- 贝叶斯优化 + TPE采样，比网格搜索高效
- 自动探索最优参数组合
- 支持早停和超时控制

## 4. 设计亮点

### 4.1 多模型层次化

```
主模型（必须）    → XGBoost/LightGBM 梯度提升树
可选模型（增强）  → LSTM/Transformer 深度学习
集成增强（可选）  → Stacking 5模型组合
```

**优势**：灵活性高，可根据数据规模和计算资源选择合适方案

### 4.2 Fallback机制

```
XGBoost未安装 → LightGBM → RandomForest
PyTorch未安装 → GradientBoostingRegressor
```

**优势**：依赖缺失时自动降级，保证系统可用性

### 4.3 增量学习

```python
# 避免全量重训练
learning_rate_decay = 0.95  # 每次更新学习率衰减
warm_start = True           # 利用已有模型
update_interval = 7 days    # 定期增量更新
```

**优势**：节省计算资源，模型持续适应新数据

### 4.4 版本管理

```
save_version() → 保存带时间戳的版本
rollback() → 回滚到指定版本
cleanup_old_versions() → 保留最近N个版本
```

**优势**：模型可追溯，支持回滚

### 4.5 并行优化

```python
# 特征准备并行
Parallel(n_jobs=32, backend="multiprocessing")

# 模型训练并行
n_jobs=-1  # 使用所有CPU核心
```

**优势**：大规模数据处理效率高
