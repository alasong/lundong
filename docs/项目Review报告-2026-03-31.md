### 1.2 架构问题清单

| 严重程度 | 问题 | 位置 | 影响 |
|:---:|---|---|---|
| 🔴 高 | **重复目录** | `src/strategy/` vs `src/strategies/` | 职责混乱，难以维护 |
| 🔴 高 | **重复文件** | `config.py` (2个), `backtester.py` (2个) | 代码冗余，版本混乱 |
| 🔴 高 | **重复类定义** | `Order` 类定义了2次 | 潜在冲突 |
| 🟡 中 | **入口文件过大** | `main.py` 55KB，1600+ 行 | 难以维护 |
| 🟡 中 | **空 __init__.py** | 11 个空文件 | 缺少模块文档 |
| 🟡 中 | **导入路径不一致** | `from src.data...` vs `from data...` | 循环依赖风险 |
| 🟢 低 | **模块数量过多** | strategies/ 目录 40+ 文件 | 需要子目录分类 |

### 1.3 依赖关系问题

```
发现的问题导入：
- src/strategies/version_manager.py: from src.data.database import...
- src/strategies/performance_tracker.py: from src.data.database import...
- src/strategies/archiver.py: from src.data.database import...

问题：使用了绝对路径 `src.xxx` 而非相对路径
```

3. **拆分 main.py**
   ```
   src/
   ├── main.py          # 精简入口，只保留参数解析
   ├── commands/        # 命令处理
   │   ├── train.py
   ```

### 5.2 代码改进 (优先级：中)

1. **添加类型标注**
   ```python
   # Before
   def train(self, features, model_type="xgboost"):
       ...

   # After
   def train(self, features: pd.DataFrame, model_type: str = "xgboost") -> Dict[str, Any]:
       ...
   ```

2. **统一导入风格**
   ```python
   # 统一使用相对导入
   from ..data.database import get_database  # 推荐
   # 而非
   from src.data.database import get_database  # 避免
   ```

3. **填充 __init__.py**
   ```python
   """
   策略模块

   包含：
   - 基础策略框架
   - 多种策略实现
   - 回测工具
   """
   from .base_strategy import BaseStrategy
   from .strategy_factory import StrategyFactory
   ```

### 5.3 文档改进 (优先级：低)

2. 为每个模块添加 `__init__.py` 文档
